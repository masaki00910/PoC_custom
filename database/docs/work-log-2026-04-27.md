### 3.4 二次チェック画面の設計上の重要発見
- `error_list_item_master.ca_group` (Dataverse の「グループ」想定) は実データでほぼ空欄
- 代わりに **`ca_attribute`** に「契約者 / 被保険者 / 受取人 / 指定代理請求人」が入っており、これがグルーピングキーとして機能している
- `error_list_items` の **全 4,973 件で `field_master_id` JOIN 成功** (orphan 0)
- 二次チェック UI は「`error_type` タブ × `attribute` セクション」の二段グルーピングで設計可能

```sql
-- 二次チェック画面のグルーピングクエリ例
SELECT elm.error_type, elm.attribute, elm.field_name,
       eli.error_flag, eli.first_check_recovery_value
FROM error_list_items eli
LEFT JOIN error_list_item_master elm ON eli.field_master_id = elm.id
WHERE eli.error_list_id = :target_id
ORDER BY elm.error_type, elm.attribute NULLS LAST, elm.field_name;
```

## 6. 既知の懸案・注意点

- **Python 3.14 の Alembic は alembic.ini を locale (cp932) で読む**: 日本語コメントを書くと UnicodeDecodeError。alembic.ini は ASCII のみ。env.py は UTF-8 OK。
- **`PYTHONIOENCODING` 未設定だと alembic ログが化ける**: 動作上は問題ないが、気になるなら `$env:PYTHONIOENCODING="utf-8"` を設定。
- **Dataverse PrimaryName ≠ UNIQUE**: `legacy_id` / `prompt_code` で実データに重複あり。新規 master を追加する時は UNIQUE を初期は付けない方が無難。
- **`error_list_items` 564 行の orphan 除外**: `ca_document_id` が空文字の行は親 `error_lists` に対応しないためロード時除外。商用移行時に業務確認すべき項目。
- **マイグレーションを編集して再 upgrade したい時**: 必ず `downgrade base` を先に走らせる。今は downgrade を `IF EXISTS` 化したので部分状態でも安全。
